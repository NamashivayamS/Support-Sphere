================================================================================
                              ARCHITECTURE OVERVIEW
================================================================================

BEFORE (Monolithic):
--------------------
Single Flask application running on one server
- All components tightly coupled
- Single point of failure
- Difficult to scale
- No redundancy

AFTER (Microservices):
----------------------
Distributed architecture with independent services
- Frontend: Nginx (3 replicas for high availability)
- Backend: Flask (1 replica, can be scaled)
- Service discovery via Kubernetes DNS
- Load balancing across frontend replicas
- Self-healing capability
- Independent scaling
- Production-ready

REQUEST FLOW:
-------------
User → NodePort (30080) → Frontend Pods (Nginx) → Backend Service (ClusterIP)
                                                 → Backend Pod (Flask)
                                                 → SQLite Database

OPTION 3: KUBERNETES (PRODUCTION)
----------------------------------
1. Build images:
   docker build -t <username>/support-backend:v1 ./backend
   docker build -t <username>/support-frontend:v1 ./frontend

2. Push to Docker Hub:
   docker push <username>/support-backend:v1
   docker push <username>/support-frontend:v1

3. Update k8s/*.yaml files with your Docker Hub username

4. Deploy:
   kubectl apply -f k8s/

5. Access:
   http://localhost:30080 (Docker Desktop)
   http://<minikube-ip>:30080 (Minikube)

================================================================================
                              TROUBLESHOOTING
================================================================================

ISSUE: Port already in use
SOLUTION: docker-compose down

ISSUE: ImagePullBackOff in Kubernetes
SOLUTION: Verify image exists on Docker Hub
          Check image name in YAML files

ISSUE: CrashLoopBackOff
SOLUTION: kubectl logs <pod-name>
          kubectl logs <pod-name> --previous

ISSUE: Backend connection failed
SOLUTION: kubectl get service backend-service
          kubectl exec -it <frontend-pod> -- wget -O- http://backend-service:5000

For more troubleshooting, see:
- DEPLOYMENT_GUIDE.md → "Common Issues and Fixes"
- QUICK_REFERENCE.md → "Quick Troubleshooting"
